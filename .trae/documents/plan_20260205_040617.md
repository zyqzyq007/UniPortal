# Backend System Development Plan

I will develop a complete backend system in a new `server/` directory within your project. This system will use **Node.js + Express** with **TypeScript**, **PostgreSQL** (via Prisma ORM), and **Jest** for testing.

## 1. Project Initialization & Architecture
- **Create `server` directory**: Isolate backend code from the frontend.
- **Setup TypeScript**: Configure `tsconfig.json` for Node.js.
- **Dependencies**: Install `express`, `cors`, `cookie-parser`, `jsonwebtoken`, `bcryptjs`, `prisma`.
- **Dev Tools**: `nodemon` (or `ts-node-dev`), `jest`, `supertest`.

## 2. Database Design (PostgreSQL + Prisma)
- **User Model**:
  - `id` (PK)
  - `username` (Unique Index)
  - `password` (Hashed)
  - `last_login` (DateTime)
  - `created_at` (DateTime)
- **Project Model**:
  - `project_id` (PK)
  - `name` (String)
  - `description` (String, optional)
  - `owner_id` (FK -> User.id)
  - `path` (String, to store file path on disk)
  - `created_at` (DateTime)

## 3. Core Modules Implementation

### A. User Authentication Module
- **POST `/api/auth/register`**: (Bonus) To create users for testing.
- **POST `/api/auth/login`**: 
  - Validate credentials.
  - Return JWT and set HTTP-only cookie.
  - Handle 404 (User not found) and 401 (Wrong password) as requested.
- **GET `/api/auth/me`**: Auto-login functionality (verify JWT).

### B. Project Management Module
- **Middleware**: `authenticateToken` to protect routes.
- **POST `/api/projects`**: Create project, auto-link `owner_id`.
- **GET `/api/projects`**: List only projects belonging to `current_user`.
- **GET/PUT/DELETE `/api/projects/:id`**: Verify `owner_id` matches `current_user` (403 if not).

### C. File Structure Parsing Module
- **GET `/api/projects/:id/structure`**:
  - Verify ownership.
  - Locate project path on disk.
  - **Service**: Recursive directory scanner.
    - Parse `package.json` and `.gitignore`.
    - Build tree JSON structure.
  - **Optimization**: Implement simple in-memory caching (LRU) or file-based caching for performance.

## 4. Testing & Documentation
- **Unit Tests**: Use Jest + Supertest to achieve >90% coverage.
  - Test Auth flows (Login success/fail).
  - Test Project isolation (User A cannot see User B's projects).
  - Test Structure parsing (mock file system).
- **API Docs**: Generate basic OpenAPI/Swagger documentation.

## 5. Execution Steps
1.  Initialize project structure and install dependencies.
2.  Define and apply Database Schema.
3.  Implement Auth API and Middleware.
4.  Implement Project CRUD API.
5.  Implement File Structure Parsing Service.
6.  Write and run Tests.
