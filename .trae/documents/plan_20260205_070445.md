# Analysis and Fix for 404 Error in Project Loading

The error `Failed to load project {code: 404, message: Project not found, data: Object}` indicates that the frontend is successfully reaching the backend API, but the backend cannot find the project with the ID provided in the URL.

## Root Cause Analysis
1.  **Route Parameter Mismatch**:
    *   In `src/router/index.ts`, the route is defined as `/projects/:projectId` (line 42).
    *   In `src/pages/projects/ProjectHome.vue` (from previous turn), the code uses `const projectId = computed(() => route.params.id as string)`.
    *   **Mismatch**: The router provides `projectId`, but the component tries to access `id`. This results in `undefined` being sent to the API, or if the component logic falls back/defaults incorrectly, it sends a wrong ID.
    *   *Correction*: The component code I reviewed in the last turn actually showed `route.params.id` in my `SearchReplace`, but I need to be 100% sure. The router definitely uses `projectId`.

2.  **Data Consistency**:
    *   The backend database was freshly initialized (SQLite `dev.db`), so it has no projects.
    *   If the user is clicking a project from `ProjectsList`, and `ProjectsList` was fetching from the *real* API, the list should be empty. If `ProjectsList` was cached or still using mock data (which I removed), it might show old IDs.
    *   However, since I refactored `ProjectsList` to use the real API, it should be showing only projects from the DB.
    *   Most likely cause: The ID passed to the API is `undefined` because of the `route.params` mismatch. `undefined` stringified might be causing the 404, or the API call is malformed.
