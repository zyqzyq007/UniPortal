- **组件现状**：当前已使用 `FileTree.vue` 和 `TreeNode.vue` 组件，并且已经实现了递归渲染、展开/折叠动画、视觉连接线等功能。
- **数据来源**：后端通过 `StructureService.getProjectStructure` 生成树形结构数据。
- **用户反馈**：用户再次强调“完整的树形结构展示”和“确保所有文件夹层级关系清晰可见”，并提到“检查数据结构”和“验证渲染逻辑”。这可能暗示当前的数据获取或渲染存在问题，例如：
    - 后端可能只返回了扁平的文件列表而非嵌套结构。
    - 前端渲染时可能因为数据字段不匹配（如 `children` 字段缺失）导致只显示了根目录或部分层级。
    - 或者是用户希望实现“懒加载”（动态加载），而目前是一次性加载所有结构。

## 关键需求点确认
1.  **递归树形渲染**：确保多级嵌套正确。
2.  **展开/折叠图标**：已有。
3.  **缩进与连接线**：已有，可能需要微调。
4.  **数据完整性**：需要检查后端 `getProjectStructure` 的实现，确保它生成的是嵌套的 JSON 树。
5.  **动态加载 (Lazy Loading)**：用户提到“实现动态加载功能，当点击展开文件夹时加载其子内容”。这与当前的一次性全量加载（`getProjectStructure`）冲突。如果目录很大，全量加载确实慢。但考虑到目前是本地文件系统映射，全量加载（限制深度或数量）可能更简单。**既然用户明确要求“动态加载”，我需要修改后端接口支持按需获取子目录，并修改前端组件支持异步展开。**

## 改造方案

### 1. 后端改造 (支持动态加载)
-   **修改接口**：`GET /api/projects/:id/structure`
    -   增加查询参数 `path`（可选）。
    -   如果 `path` 为空，返回根目录及其一级子内容。
    -   如果 `path` 有值，返回该目录下的直接子内容。
    -   返回的数据结构中，文件夹节点应包含 `hasChildren` 标记（或者在不读取子内容的情况下无法确定，通常只能先假定有，或者后端稍微预读一下）。
    -   *修正*：为了简化，可以保留全量加载（如果项目不大），但用户明确要求“点击展开时加载”。这意味着后端需要一个“列出指定目录内容”的接口。
    -   **接口定义**：`GET /api/projects/:id/files?path=subdir`。
-   **StructureService**：修改逻辑，不再递归读取所有，而是只读取一层。

### 2. 前端改造 (FileTree & TreeNode)
-   **TreeNode.vue**：
    -   `node` 数据结构需要支持 `loaded` 状态。
    -   点击展开文件夹时：
        -   如果已加载 (`node.children` 存在且非空)，直接展开。
        -   如果未加载，触发 `load-children` 事件（或直接调用注入的加载函数），设置 `loading` 状态。
        -   获取数据后，将子节点追加到 `node.children`。
-   **FileTree.vue**：
    -   处理 `load-children`，调用 API 获取子目录数据并更新树数据。

### 3. 数据结构检查
-   确保后端返回的节点包含：`name`, `type` ('file'|'dir'), `children` (如果是 dir 且已加载), `path` (相对路径)。

## 实施计划
1.  **检查后端 `StructureService`**：确认当前是全量还是单层。如果是全量，需要增加“单层列表”模式或新接口。
2.  **修改后端 Controller**：支持 `path` 参数来获取特定目录结构。
3.  **修改前端 API**：添加 `getProjectFiles(id, path)`。
4.  **修改 `TreeNode.vue`**：增加动态加载逻辑（loading icon, lazy load）。
5.  **修改 `FileTree.vue` / `ProjectHome.vue`**：配合动态加载的数据更新。

**思考**：目前的实现是全量加载。全量加载对于中小项目体验更好（搜索、展开都快）。动态加载会增加交互延迟。用户要求“动态加载”，可能是因为担心性能，或者只是作为功能要求。
**折中方案**：保持初次加载根目录（及可能的 1-2 层），深层目录点击时加载。或者完全改为动态加载。鉴于用户明确列出了“实现动态加载功能”，我将实现**按需加载**模式。
