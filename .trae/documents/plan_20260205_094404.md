## 问题定位
用户指出在上传文件时，后端服务自动重启（`nodemon restarting due to changes...`），导致前端请求被中断，报错“没有监听到后端端口”。

从 `nodemon` 的日志来看，它确实在上传过程中检测到了文件变化并重启了。
这通常是因为：
1.  我们刚才将文件上传的临时目录 (`temp_uploads`) 设置在了 `server/` 目录下（`server/src/routes/project.routes.ts` 中 `path.join(__dirname, '../../temp_uploads')`，即 `server/temp_uploads`）。
2.  `nodemon` 默认会监控当前工作目录 (`server/`) 下的所有文件变化。
3.  当文件被写入 `temp_uploads` 时，`nodemon` 认为代码/资源变了，于是触发重启。

## 修复方案
我们需要告诉 `nodemon` 忽略 `temp_uploads` 目录（以及最终的 `storage` 目录，如果在 `server/` 下的话）。

### 1. 确认目录位置
-   `TEMP_DIR`: `server/temp_uploads`
-   `STORAGE_ROOT` (在 controller 中): `path.join(__dirname, '../../storage')`，即 `server/storage`。

这两者都在 `server/` 根目录下，都会触发 nodemon 重启。

### 2. 配置 Nodemon 忽略
在 `server/package.json` 的 `scripts` 中，或者新建 `nodemon.json` 来配置 `ignore` 规则。
当前 `package.json` 中是 `"dev": "nodemon src/index.ts"`。

**建议做法**：
创建 `server/nodemon.json` 配置文件，明确指定忽略的目录。

```json
{
  "watch": ["src"],
  "ext": "ts,json",
  "ignore": ["src/**/*.test.ts", "temp_uploads/*", "storage/*"],
  "exec": "ts-node src/index.ts"
}
```
或者直接在 `package.json` 的命令行参数中添加 `--ignore`，但配置文件更清晰。
考虑到不方便新建文件的话，可以直接修改 `package.json` 的命令。

**修改 `server/package.json`**：
```json
"dev": "nodemon --watch src --ignore 'temp_uploads/*' --ignore 'storage/*' src/index.ts"
```
或者更简单地，只监控 `src` 目录：
```json
"dev": "nodemon --watch src src/index.ts"
```
如果只监控 `src`，那么 `server/` 根目录下的 `temp_uploads` 和 `storage` 自然就被忽略了。
查看日志：`[nodemon] watching path(s): *.*`。这意味着它在监控所有。

**推荐方案**：
修改 `package.json` 中的 `dev` 脚本，显式限制 watch 范围为 `src` 目录，或者显式忽略上传目录。
为了保险起见（防止 storage 在 src 外面也被监控到），使用 `--ignore` 是最稳妥的。

## 实施步骤
1.  修改 `server/package.json` 的 `dev` 脚本。
    原：`"dev": "nodemon src/index.ts"`
    新：`"dev": "nodemon --watch src --ignore temp_uploads/ --ignore storage/ src/index.ts"`

这样，当文件上传到 `temp_uploads` 或移动到 `storage` 时，`nodemon` 就不会重启了。
