## 需求分析
用户希望在 Web 端实现类似 VS Code 的专业代码编辑与可视化体验。核心需求包括：
1.  **专业编辑器**：语法高亮、智能提示、高性能（支持万行代码）。
2.  **代码结构可视化**：展示代码的结构（类、函数、变量）和调用关系。
3.  **智能体验**：文档注释、快捷键、鼠标悬停提示。

## 技术方案
鉴于 VS Code 的核心编辑器 **Monaco Editor** 开源且支持 Web，这是最佳选择。

### 1. 核心编辑器集成 (Monaco Editor)
-   **引入库**：`monaco-editor`。
-   **功能**：
    -   自动语法高亮（支持 JS, TS, Python, Vue, HTML, CSS 等主流语言）。
    -   MiniMap（代码缩略图）。
    -   Hover 提示、代码折叠、快捷键（Ctrl+F 搜索等）。
    -   高性能渲染（基于 Canvas/DOM 混合，轻松处理 10000+ 行）。

### 2. 代码结构可视化 (Outline & Structure)
-   **实现方式**：利用 Monaco 的 `executeDocumentSymbolProvider` 命令获取当前文件的 AST 符号信息（Symbols）。
-   **可视化组件**：
    -   开发一个 **CodeStructure** 组件，显示在编辑器右侧或作为浮层。
    -   使用树状图展示 Class -> Method -> Variable 层级。
    -   点击节点可跳转到对应代码行。
    -   对于 JS/TS，Monaco 内置了较好的分析能力；对于其他语言，提供基础的正则或缩进分析作为降级方案。

### 3. 调用层级与智能注释
-   **调用层级**：集成 Monaco 原生的 "Peek Call Hierarchy" 功能（需语言服务支持）。
-   **智能注释**：配置 Snippets，输入 `/**` 自动触发文档注释模板。

### 4. 工程化配置
-   **Vite 配置**：Monaco 需要加载 Web Workers 来处理语法分析，需要配置 Vite 插件或手动处理 Worker 路径。

## 实施步骤
1.  **安装依赖**：`npm install monaco-editor`。
2.  **Vite 配置**：配置 Monaco Workers 的构建与加载。
3.  **组件开发**：
    -   `CodeEditor.vue`: 封装 Monaco 实例，处理 props（内容、语言、主题）与 resize 事件。
    -   `CodeStructure.vue`: 获取并渲染代码结构树。
4.  **集成**：修改 `EditorPane.vue`，用 `CodeEditor` 替换原有的文本展示区，并并入结构视图。
5.  **验证**：测试大文件加载性能与结构解析准确性。

## 风险评估
-   **Python 支持**：Monaco 对 Python 的**结构分析**（Outline）在纯前端环境下可能较弱（通常需要后端 LSP）。如果内置分析不够，我将实现一个简易的 Python AST 正则解析器来辅助生成结构树。

我将优先保证 JS/TS 的完美体验，并尽可能覆盖 Python。
