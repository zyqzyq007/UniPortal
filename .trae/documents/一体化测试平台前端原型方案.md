## 现状结论

* 仓库当前仅有 PPT 文件，没有任何 Vue 工程或现有前端代码可复用，需要从零创建原型工程与页面结构。

## 目标理解

* 基于需求文档实现 MVP 前端原型，闭环流程为：登录 → 选项目 → 选工具 → 创建任务 → 任务列表 → 任务详情 → 查看结果。

* 覆盖 11 类页面（含可选结果页），并具备路由守卫与未登录拦截。

## 技术选型与约定

* 使用 Vue 3 + Vite + Vue Router + TypeScript。

* UI 采用基础组件与简洁布局（不引入额外 UI 库，确保仓库内依赖可控）（主题颜色采取深蓝色）。

* 使用本地模拟数据与最小状态管理（必要时引入 Pinia）。

## 目录与路由设计

* 创建 `src/` 结构：`router/`、`layouts/`、`pages/`、`components/`、`mock/`。

* 路由以 `projectId` 为上下文，覆盖登录、项目、工具、任务、结果等路径。

* 全局壳包含顶部用户信息与侧边栏（工具中心、任务中心）。

## 页面与功能拆分

* 登录页：登录态写入 + 跳转 `/projects`，失败提示占位。

* 项目列表页：展示模拟项目，进入 `/projects/:projectId`。

* 项目详情壳：仅做入口导航。

* 工具中心页：6 工具卡片，进入对应创建页。

* 6 个工具创建页：表单 → 提交 → 成功提示 → 跳转任务详情。

* 任务中心页：任务列表（最小字段集）。

* 任务详情页：任务信息 + 结果入口 + 失败信息占位。

* 结果查看页（可选）：提供 iframe/跳转/下载占位。

* 404/无权限页：基础错误处理与路由守卫落地。

## 路由守卫与权限

* 未登录访问主功能页面时自动跳转登录页。

* 登录态失效后回跳登录页（前端模拟）。

## 验证方式

* 启动本地开发服务器，逐页访问并验证路由与跳转闭环。
