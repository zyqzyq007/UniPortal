## 改造目标
1) 选中文件时能够获得 **完整路径 + 文件类型**。
2) 通过 API 请求加载文件内容，支持文本预览，不支持二进制提示。
3) 编辑器区域增加 loading/错误处理、快速切换防抖、安全清空。

## 方案与改动点
### 1. 事件携带信息
- 修改 [TreeNode.vue](file:///Users/bytedance/kairos/tasks/UniPortal/src/components/TreeNode.vue)
  - `emit('open-file', { path, type })`，增加 `type` 字段。
- 修改 [FileTree.vue](file:///Users/bytedance/kairos/tasks/UniPortal/src/components/FileTree.vue)
  - `defineEmits` 改为 `{ path, type }`。

### 2. 新增后端文件内容 API
- 新增接口：`GET /api/projects/:id/file?path=...`
  - 读取项目目录下的文件
  - 仅允许位于项目根目录内的路径（防止路径穿透）
  - 文本文件直接返回内容
  - 二进制返回 `{ code: 415, message: '不支持预览' }`
- 代码位置：后端 [project.controller.ts](file:///Users/bytedance/kairos/tasks/UniPortal/server/src/controllers/project.controller.ts) + [project.routes.ts](file:///Users/bytedance/kairos/tasks/UniPortal/server/src/routes/project.routes.ts)

### 3. 前端 API 封装
- 在 [src/api/projects.ts](file:///Users/bytedance/kairos/tasks/UniPortal/src/api/projects.ts) 增加 `getProjectFileContent(id, path)`。

### 4. 编辑器显示逻辑
- 修改 [EditorPane.vue](file:///Users/bytedance/kairos/tasks/UniPortal/src/components/EditorPane.vue)
  - 新增 props: `fileType`/`filePath`
  - 监听 filePath 变化：
    - 立即清空编辑器内容
    - 设置 loading 状态
    - 调用 API 获取内容
    - 二进制返回显示“当前文件不支持预览”
    - 错误显示“加载失败”并保留重试按钮

### 5. ProjectHome 层级联动
- 在 [ProjectHome.vue](file:///Users/bytedance/kairos/tasks/UniPortal/src/pages/projects/ProjectHome.vue)
  - `onOpenFile` 改为接收 `{ path, type }`
  - 将选中信息传给 `EditorPane`
